# Production Pipeline

## Purpose

Create a reproducible system protecting canon across ChatGPT, Codex, ComfyUI, and future tools.

## Division

- **Creative control:** canon, narrative logic, pacing, identities, world and manifestation design.
- **Production execution:** structured files, prompts, datasets, validators, references, ComfyUI, LoRA, and reproducible changes.
- **Prompt spawning:** load only current system rules, relevant canon/references, and the scene brief.

## Priority

1. Lock rendering language.
2. Begin with character design in ComfyUI.
3. Keep prompt-driven FLUX generation core.
4. Use one strong canonical identity reference, not chained references.
5. Replicate style/workflow before identity packs.
6. Stabilize characters and manifestations.
7. Expand into environments, architecture, cinematography, action, expressions, and comics.

## Candidate LoRA ecosystem

**Proposal:** AA_Render, AA_Characters, AA_Manifestations, AA_Fractals, AA_Architecture, AA_Environment, AA_Cinematography, AA_Action, AA_Expressions, AA_Comic. LoRAs condition one generation simultaneously; they are not separately composited renders. Individual character LoRAs are not automatically first priority.

## Workflows

### Character
Load render language and identity → use strongest reference → portrait → revise/approve → name if new → sheet → stress-test → register canon.

### Manifestation
Define user, medium, motion, bodily integration, local environment influence, functional range, maximum, loss of control, exhaustion, and residue → ensure progression increases specificity → validate.

### Environment
Define geography, climate, culture, history, population, materials, architecture, environmental evidence, lighting, atmosphere, and any Intentional Dissonance → generate broad views before decorative details → reject genre defaults.

### Comic
Lock purpose → break into beats without unnecessary subchapters → give beats distinct visual functions → track shot/angle/pose/location/action/emotion → prevent repetition → generate text-free art unless requested → letter separately → validate continuity, anatomy, pacing, and bubbles → await approval before advancing.

## Change management

Every canon change updates its canon file, the registry, affected production rules, and the changelog. Never erase historical candidate or deprecated status silently.
